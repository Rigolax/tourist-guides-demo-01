# Статическая версия сайта (GitHub Pages)

**Назначение:** временная публикация витрины без Node.js, Express, SQLite и админки.  
**Основной проект** с бэкендом лежит выше: `tourist-guides/` (`server/`, `public/` и т.д.) — его не трогайте.

## Что публиковать

Загружайте на GitHub Pages **содержимое этой папки** (`gh-pages-version/`), т.е. файлы должны оказаться в **корне** сайта:

- `index.html`, `guide.html`, `attr-*.html`, `mc-*.html`
- `styles.css`
- папки `assets/`, `data/`, `js/`

Варианты:

1. Ветка **`gh-pages`**: скопируйте **внутреннее** содержимое `gh-pages-version/` в корень ветки (не саму папку целиком).
2. **Actions / деплой:** артефакт — те же файлы в корне опубликованного сайта.
3. Локально: `npx serve .` из каталога `gh-pages-version` (или любой статический сервер).

## Важно

- Открытие **`file://`** часто ломает `fetch("data/guides.json")` — используйте локальный HTTP-сервер или GitHub Pages.
- Карта «Музеи» без ключа API и без `/api/config` показывает подсказку; это ожидаемо для статики.
- Админка (`/admin`) в эту сборку **не входит**.
- Гиды читаются из **`data/guides.json`**; скрипты: `js/index-static.js`, `js/guide-static.js` (не путать с `public/js/index.js` в основном проекте).

## Сайт в подкаталоге репозитория

Если URL вида `https://user.github.io/ИМЯ-РЕПО/` и GitHub отдаёт сайт **не из корня**, а из `gh-pages-version/`, добавьте во все HTML в `<head>`:

```html
<base href="/ИМЯ-РЕПО/gh-pages-version/">
```

(путь уточните в настройках Pages). Сейчас ссылки **относительные** и рассчитаны на корень сайта.

## Файл `.nojekyll`

В корне лежит `.nojekyll` — чтобы GitHub Pages не прогонял сайт через Jekyll.
